# AI Agent 工程学习清单(第二阶段起)

> 定位:Python 基础已过,正式进入 Agent 工程。目标是能独立做出可上线的 Agent,拿到相关 offer。
> 用法:按阶段顺序学,**别跳第二阶段**。每项配「验收标准」,能做出来就过。
> **核心原则:越往后越要「做项目」而不是「记知识点」。3 个能跑的 Agent 项目 > 10 份教程笔记。**
> 卡住了把报错 / 代码贴给 Claude Code 问。

---

## 前置检查:第一阶段(Python 基础)✅

进入第二阶段前,确认这些你已经能不看教程写出来:

- [x] async / await、`asyncio.gather` 并发
- [x] `httpx` 调 API、读 HTTP 状态码
- [x] Pydantic 定义模型、校验、`model_validate`
- [x] try / except 错误处理 + 重试(tenacity)
- [x] `.env` + pydantic-settings 管配置

---

## 第二阶段:手写裸 Agent(不用框架)⭐ 最关键

- [ ] 用 `httpx` 直接调大模型 API(Claude / OpenAI 的 chat completions)
- [ ] 理解 **messages 数组**:system / user / assistant 三种角色怎么组织对话
- [ ] 定义工具:用 Pydantic 描述工具的参数 schema(第 7 步那个「杀手级用法」)
- [ ] **手写工具调用循环**(Agent 的心脏):
      模型说要调工具 → 代码执行 → 结果塞回 messages → 模型继续 → 直到模型说「完成」
- [ ] 加错误处理:工具执行失败、模型返回格式不对时不崩

✅ **验收**:不用任何 Agent 框架,手写一个能调 2~3 个工具(如查天气、算数学)的循环,
模型能自己决定调哪个、何时停止。

---

## 第三阶段:MCP(Model Context Protocol)

- [ ] 理解 MCP 解决什么:**给 Agent 接工具/数据源的标准协议**,不用每个工具重写对接
- [ ] 用现成的 MCP server(文件系统、数据库、搜索)
- [ ] 自己写一个简单的 MCP server 暴露一个工具

✅ **验收**:让你的 Agent 通过 MCP 接一个外部工具,跑通。

---

## 第四阶段:RAG + 记忆(2~3 周)

- [ ] **Embedding(向量化)** 是什么、**向量数据库**(Chroma / Qdrant)怎么用
- [ ] **RAG 流程**:文档切块 → 向量化 → 检索 → 塞进 prompt
- [ ] **记忆**:短期(对话历史)vs 长期(跨会话持久化)

✅ **验收**:做一个能回答「你自己文档内容」的问答 Agent(喂一批文档,它基于文档回答)。

---

## 第五阶段:Agent 框架(主攻 LangGraph)(2~3 周)

- [ ] **LangGraph**(生产默认首选,状态机,招聘需求最大)—— 重点
- [ ] 了解 CrewAI(多角色团队,原型快)、Claude Agent SDK(工具调用轻量)存在即可
- [ ] 用 LangGraph 把第二阶段手写的 Agent 重构一遍,体会框架帮你管了哪些状态

✅ **验收**:用 LangGraph 做一个带状态、多步骤的 Agent(如带审批环节的流程)。

---

## 第六阶段:生产化 + 作品集(持续)⭐ 从「会写 demo」到「拿 offer」的分水岭


- [ ] **评估(eval)**:怎么衡量 Agent 好不好用,而不是靠感觉
- [ ] **可观测性**:LangSmith / tracing,看清 Agent 每一步在干嘛
- [ ] **可靠性**:超时、重试、降级、防无限循环(第一阶段的重试用上)
- [ ] **成本控制**:统计 token 花了多少

✅ **验收**:把某个 Agent 做成**完整的、放 GitHub 的项目**:有 README、配置管理、基本测试。
这就是你的面试作品。
